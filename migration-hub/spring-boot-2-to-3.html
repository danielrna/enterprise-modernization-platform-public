Try in 5 Minutes

Run a readiness report on any Java or Spring repository. The output is a static HTML report and a machine-readable JSON report.

docker run --rm -v "$PWD:/workspace" danielrna/enterprise-modernization-platform:v0.1.0 analyze . --pack spring-boot-3-readiness --out reports/spring-boot-readiness

Open the generated report:

reports/spring-boot-readiness/index.html

GitHub Action

name: Spring Boot Readiness

on:
  workflow_dispatch:
  pull_request:

jobs:
  readiness:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run readiness analysis
        uses: danielrna/enterprise-modernization-platform-public@v0.1.0
        with:
          path: .
          pack: spring-boot-3-readiness
          out: emp-report

      - name: Upload report
        uses: actions/upload-artifact@v4
        with:
          name: emp-readiness-report
          path: emp-report

Guide

This guide focuses on readiness evidence: build metadata, Spring Boot version detection, Jakarta namespace risk, static source findings, and an HTML report that can be shared without a backend.

Readiness Signals

  • Explicit Maven or Gradle build metadata
  • Spring Boot version discovery
  • javax to jakarta migration risk
  • Dependency injection and legacy API cleanup findings
  • OpenRewrite dry-run readiness for the transformation flow

Risk Matrix

What This Does Not Do

  • It does not run a hosted SaaS backend.
  • It does not parse Java with a custom compiler or AST.
  • It does not replace project-specific migration review.
  • It does not apply changes during readiness analysis.

FAQ

Does readiness analysis apply migrations? No. It measures readiness and creates evidence. Apply, rollback, compile, and test validation are part of the transformation flow.

Can this run in CI? Yes. Use the Docker image directly or the GitHub Action. The output directory can be uploaded as a build artifact.

Does it require a server? No. Reports, benchmark pages, and this hub are static files.
