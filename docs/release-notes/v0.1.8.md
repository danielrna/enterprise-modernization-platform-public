# v0.1.8

Spring Security 6 readiness pack with security migration findings, Knowledge Base guidance, and first public benchmark evidence.

## Spring Security 6 readiness pack

Add a dedicated readiness pack for Spring Security 6 migration planning.

Audience: Consultants and enterprise Java teams planning Spring Security 5.x to 6.x migrations

- Adds `packs/spring-security-6-readiness.json`.
- Detects Spring Security usage and explicit Spring Security 5.x versions when metadata is available.
- Finds WebSecurityConfigurerAdapter, legacy matcher APIs, authorizeRequests configuration, and global method security annotations.
- Adds report next actions for Spring Security 6 configuration risks.

Links:
- [Spring Security 6 pack docs](docs/packs/spring-security-6-readiness.html)
- [Spring Security 6 Knowledge Base](docs/knowledge-base/spring-security-6-readiness.html)

## Spring Security benchmark evidence

Add the first catalog-backed Spring Security readiness benchmark batch.

Audience: Consultants building public proof for security modernization work

- Adds five Spring Security readiness reports.
- Raises the public benchmark catalog from 60 to 65 reports.
- Keeps checkout-backed evidence unchanged while establishing the public report shape for the new pack.

Links:
- [Benchmark index](docs/benchmarks/index.html)
- [Spring Security Samples Legacy report](docs/benchmarks/spring-security-samples-legacy/index.html)

## Spring Security 6 Knowledge Base

Add Knowledge Base guidance for interpreting Spring Security 6 readiness findings.

Audience: Consultants explaining Spring Security migration risk

- Adds `knowledge/spring-security-6-readiness.json`.
- Documents risks around removed adapters, matcher APIs, authorization DSL changes, method security, and Jakarta overlap.
- Clarifies validation limits for security behavior.

Links:
- [Spring Security 6 Knowledge Base](docs/knowledge-base/spring-security-6-readiness.html)
